# ⚙️ 공통 코딩 규칙 (Config)

팀원 모두가 한 사람이 짠 것처럼 일관된 코드를 작성할 수 있도록 돕는 **프로젝트 환경 설정 파일** 모음입니다.

**무엇이 들어가나요?**
- **ESLint / Prettier 설정:** 들여쓰기 간격, 따옴표 규칙 등 코드 스타일 자동화 규칙
- **TypeScript 설정 (`tsconfig`):** 타입 검사 엄격도 등 컴파일 규칙

**왜 필요한가요?**
각 프로젝트 폴더마다 설정 파일을 따로 만들 필요 없이, 이 폴더의 규칙 하나만 상속받아 사용하면 프로젝트 전체의 코드 퀄리티를 깔끔하게 통일할 수 있습니다.
