# 📝 공통 데이터 규격 (Types)

프론트엔드와 백엔드가 데이터를 주고받을 때 서로 오해가 없도록 정의한 **TypeScript 타입(인터페이스) 계약서** 모음입니다.

**무엇이 들어가나요?**
- Lighthouse 측정 결과 JSON 스키마 (`LCP`, `CLS` 등)
- 서버에서 보내주는 통신사/지역별 지연시간 데이터 구조
- API 응답 성공/실패 공통 규격

**우리 프로젝트 적용 예시:**
- **Lighthouse 점수 규격:** AI 파이프라인이나 자동화 봇이 측정한 `LCP`, `CLS` 점수를 대시보드에 띄울 때, 데이터가 어떤 구조(숫자인지, 문자열인지)로 생겼는지 미리 정의해 둡니다. 
(`type LighthouseScore = { LCP: number; CLS: number; ... }`)
- **통신사별 딜레이 데이터:** 백엔드에서 넘겨주는 SKT, KT, LGU+ 통신사별 지연 시간 데이터의 형태를 정의합니다.

**사용 방법:**
백엔드에서 데이터를 받을 때나 프론트엔드에서 데이터를 화면에 뿌려줄 때, 이곳에 정의된 타입을 가져다 사용하여 에러를 사전에 방지합니다.
